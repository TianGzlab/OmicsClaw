## 为什么需要明确的安全规则

基因组数据、单细胞测序结果、临床代谢组数据均属于高度敏感的生物医学信息，涉及个人隐私乃至遗传特征，任何未经授权的外发或滥用都可能造成不可逆的伤害。

AI 工具天然倾向于输出"听起来合理"的结论，而非经过严格验证的科学结果。不对方法论加以硬性约束，模型就可能臆造参数、阈值或基因关联，导致分析结论无法复现甚至误导决策。

OmicsClaw 是研究与教育工具，不具备医疗器械资质，其输出不构成任何临床诊断或治疗建议。明确的安全规则确保每一份报告都清楚告知读者这一边界。

## 4 条核心规则（必读）

### 1. 基因数据永不离开本机

所有分析均在本地运行，任何原始数据、中间结果或最终报告均不会上传至外部服务器。

OmicsClaw 的 skill 脚本直接调用本机 Python 环境，不依赖云端计算或远程 API 传输数据。无论是 h5ad 文件、计数矩阵还是 VCF 文件，全程保留在用户指定的本地目录中。即使使用 LLM 辅助路由，传入 API 的也仅是用户的文本指令，而非底层组学数据。

### 2. 每份报告必含免责声明

每次 skill 执行生成报告时，报告文件的顶部或页脚必须嵌入以下免责声明：

> "OmicsClaw 是用于多组学分析的研究与教育工具。它不是医疗器械，不提供临床诊断。在基于这些结果做决定前请咨询领域专家。"

该声明在各 skill 的 `report.md` 生成逻辑中硬编码为必填项，不可被用户配置跳过。这条规则来源于 CLAUDE.md Safety Rules 第 2 条，原文为："*OmicsClaw is a research and educational tool for multi-omics analysis. It is not a medical device and does not provide clinical diagnoses. Consult a domain expert before making decisions based on these results.*"

### 3. 严守 SKILL.md 方法论

每个分析决策——参数选择、过滤阈值、基因关联——都必须追溯到对应 skill 的 `SKILL.md` 文档中的具体段落。

禁止凭经验猜测或模型先验知识填写生物信息学参数。若某个 skill 的 `SKILL.md` 未覆盖某项操作，agent 应拒绝执行并明确告知用户，而非以"合理估计"代替。所有方法学决策均需可审计、可复现。

### 4. 覆盖前警告

在向已存在内容的输出目录写入任何文件之前，agent 必须主动向用户告知并请求确认。

这一规则防止静默覆盖已完成的分析结果。检测到目标目录非空时，OmicsClaw 会暂停并列出将被覆盖的文件，等待用户明确同意后方可继续。

## 启用前 / 启用后

| 维度 | 无明确规则 | 有规则 |
|---|---|---|
| 数据外发风险 | 取决于工具实现 | 强制本地处理 |
| 报告法律责任 | 性质模糊 | 明示研究/教育用途 |
| 参数臆造 | 可能发生 | 强约束于 SKILL.md |

## 触发警告的情况

- **用户询问的基因 / 通路在 SKILL.md 没覆盖**：agent 会拒绝臆造，明确告知该操作未被任何 skill 涵盖
- **skill 输出目录非空**：覆盖前暂停并提示（规则 4）
- **用户上传图像含可识别个人信息**：bot 会在分析前提醒并要求用户确认
- **请求执行 skill 范围外的操作**（如修改 OmicsClaw 源代码）：拒绝并解释超出 agent 职责边界
