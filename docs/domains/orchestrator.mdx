## 本领域解决什么问题

Orchestrator 是元领域（meta-domain），不执行分析，提供两项支撑：orchestrator 将自然语言路由到正确的分析 skill；omics-skill-builder 生成新 skill 脚手架。列为独立 domain 是为保持注册表对称性，让两个入口在 `omicsclaw.py list` 可见。路由原理见 [架构 · 编排器](/docs/architecture/orchestrator)。

## 典型使用流程 ASCII 图

```text
开发者新增 skill：
   omics-skill-builder
        │
        ▼
   生成 skills/<domain>/<new-skill>/{SKILL.md, <slug>.py}
        │
        ▼
   写入实现 + commit

用户跑分析：
   自然语言提问
        │
        ▼
   orchestrator → 路由到具体 skill
```

## 何时不适用本领域

- **直接想跑分析** → 用 6 个分析领域之一，本域是元工具
- **想改 orchestrator 路由逻辑** → 看代码 `omicsclaw/routing/router.py`，本 skill 文档不覆盖内部修改
- **想改 LLM provider** → 调整 `.env`，不在本域 skill 范围内
