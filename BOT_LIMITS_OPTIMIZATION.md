# Bot Limits Optimization

## 问题

原有的 bot 存在两个限制可能影响复杂的多步骤组学分析工作流：

1. **MAX_HISTORY = 20** - 对话历史只保留 20 条消息
2. **MAX_TOOL_ITERATIONS = 10** - 工具调用最多迭代 10 次

## 优化方案

### 1. 增加对话历史限制

**之前**: MAX_HISTORY = 20
**现在**: MAX_HISTORY = 50 (可通过环境变量配置)

**原因**:
- 复杂的组学分析工作流需要更长的上下文
- 有了内存系统支持，可以安全地增加历史长度
- 重要信息会被保存到持久化内存中

### 2. 增加工具调用迭代限制

**之前**: MAX_TOOL_ITERATIONS = 10
**现在**: MAX_TOOL_ITERATIONS = 20 (可通过环境变量配置)

**原因**:
- 支持更复杂的多步骤工作流
- 例如：预处理 → 域识别 → 差异表达 → 富集分析 → 可视化
- 仍然有上限防止无限循环

### 3. 环境变量配置

新增两个可选的环境变量：

```bash
# 对话历史最大消息数（默认: 50）
OMICSCLAW_MAX_HISTORY=50

# 工具调用最大迭代次数（默认: 20）
OMICSCLAW_MAX_TOOL_ITERATIONS=20
```

## 使用场景

### 场景 1: 简单分析（默认配置足够）
```
用户: "预处理 data.h5ad"
→ 1 次工具调用
→ 2 条消息（用户 + 助手）
```

### 场景 2: 复杂工作流（需要更多迭代）
```
用户: "完整分析 data.h5ad：预处理、找域、差异表达、富集分析"
→ 可能需要 4-8 次工具调用
→ 10-20 条消息
→ 现在可以顺利完成
```

### 场景 3: 交互式探索（需要更长历史）
```
用户进行多轮对话探索数据
→ 30-50 条消息的上下文
→ 现在可以保持完整对话历史
```

## 实现细节

```python
# bot/core.py
MAX_HISTORY = int(os.getenv("OMICSCLAW_MAX_HISTORY", "50"))
MAX_TOOL_ITERATIONS = int(os.getenv("OMICSCLAW_MAX_TOOL_ITERATIONS", "20"))
```

## 向后兼容

- 默认值已优化，无需配置即可获得更好的体验
- 用户可以通过环境变量自定义限制
- 不影响现有功能

## 测试

现有的所有测试仍然通过，新的限制不会破坏任何功能。
