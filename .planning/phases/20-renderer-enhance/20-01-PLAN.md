**Function 6: plot_enrichment_network**

Enrichment gene-term network graph (EnrichmentPlot network equivalent from scop). This visualizes the relationship between enriched terms and their associated genes as a bipartite graph.

CSV input: enrichment_results.csv (already exported by sc-enrichment). Required columns: term, pvalue_adj, genes (slash-separated gene list, e.g. "TP53/BRCA1/MYC"). If genes column is missing, fall back to top_terms.csv.

Implementation approach (adapted from scop EnrichmentPlot network, but using base igraph instead of scop internals):

1. Read CSV, filter to significant terms (padj < 0.05 by default), take top N terms (default 10).
2. Build bipartite graph: term nodes + gene nodes, edges = term-gene membership.
3. Use igraph::graph_from_data_frame() for graph construction.
4. Layout with igraph::layout_with_fr() (Fruchterman-Reingold). Guard with requireNamespace("igraph") -- if igraph not available, print WARNING and quit(status=0).
5. Extract node coordinates. Term nodes = large circles colored by -log10(padj), gene nodes = small labeled rectangles.
6. Render with ggplot2: geom_segment for edges, geom_point for nodes, geom_text/ggrepel for labels.
7. Term nodes sized by gene count (number of edges). Gene nodes fixed small size.
8. Color: terms use viridis scale for -log10(padj). Gene labels in grey30.
9. Params: top_n (default 10), group (select specific cell group), padj_cutoff (default 0.05).

Gene column detection: try "genes" first, then "geneID", then "leading_edge". Split on "/" or "," or ";".

**Function 7: plot_enrichment_enrichmap**

Enrichment term similarity map (EnrichmentPlot enrichmap equivalent from scop). This clusters enriched terms by gene overlap similarity and displays them as a network where edges represent shared genes.

CSV input: enrichment_results.csv. Same columns as above plus requires the genes column.

Implementation approach:

1. Read CSV, filter significant, take top N terms (default 50 -- enrichmap benefits from more terms).
2. Build term-term similarity matrix: Jaccard similarity of gene sets between all term pairs.
3. Create graph: terms are nodes, edges where Jaccard > 0.1 (configurable via min_similarity param). Edge weight = Jaccard coefficient.
4. Use igraph for graph + layout (layout_with_fr). Guard with requireNamespace("igraph").
5. Cluster terms with igraph::cluster_fast_greedy().
6. Render with ggplot2: geom_segment for edges (alpha proportional to weight), geom_point for term nodes.
7. Node size = gene count. Node color = cluster membership (discrete palette via omics_palette). 
8. Label top 3 terms per cluster using ggrepel (requireNamespace guard, fall back to geom_text if unavailable).
9. Params: top_n (default 50), group, min_similarity (default 0.1), padj_cutoff (default 0.05).

Both functions: wrap the entire body in tryCatch with error handler that prints to stderr and quit(status=1). Use `%||%` operator for param defaults. Print "Saved: path" on success via ggsave_standard.

**Part A: Add plot_de_manhattan to de.R**

Manhattan plot for multi-group DE results (DEtestPlot manhattan equivalent from scop). This shows all groups side-by-side as vertical columns with jittered gene points colored by log2FC.

CSV input: de_top_markers.csv (same as volcano). Uses the shared .detect_de_columns() and .read_de_csv() helpers already in de.R.

Implementation approach (adapted from scop DEtestManhattanPlot):

1. Read de_top_markers.csv via .read_de_csv(). Detect columns via .detect_de_columns().
2. Require "group" column (multi-group is the point of manhattan). If missing, fall back: treat as single group "All" and plot as simple strip chart.
3. Assign integer x position per group. Jitter x within group (runif, width 0.4). Y = fold change value.
4. Build background tiles: one colored tile per group at y=0 (cell type color bar, using omics_palette).
5. Points colored by log2FC gradient: blue-white-red (scale_color_gradient2, midpoint=0).
6. Highlight significant genes (padj < threshold) with black border (geom_point outline layer).
7. Label top N genes per group (by |FC| * -log10(padj) score) using ggrepel::geom_text_repel. Guard with requireNamespace.
8. Y-axis = "Average log2FoldChange". X-axis = group labels via colored tiles at y=0.
9. Params: padj_thresh (default 0.05), fc_thresh (default 0.25), n_label (default 3), jitter_width (default 0.4).
10. Dynamic sizing: width = max(7, n_groups * 1.8 + 2), height = 7.
