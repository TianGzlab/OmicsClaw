**Part A: Enhance plot_pseudotime_dynamic in pseudotime.R for multi-lineage comparison**

The current plot_pseudotime_dynamic reads gene_expression.csv and pseudotime_points.csv, then facets by gene. The enhancement adds lineage-aware plotting: when the expression data contains a "lineage" column, each lineage gets its own loess curve overlaid on the same panel (per gene), with different colors per lineage and CI ribbons.

Modify plot_pseudotime_dynamic (do NOT create a new function -- enhance the existing one):

1. After reading gene_expression.csv (around the merge step at line ~203), detect if a "lineage" column exists in expr_df or pts_df.
2. If lineage column exists in pts_df, merge it into df_long.
3. If NO lineage column: behavior is unchanged (backward compatible). Current code path continues with single loess per gene facet.
4. If lineage column IS present:
   - In the ggplot, change the stat_smooth layer to group by lineage: add `aes(color = lineage, fill = lineage)` and `group = interaction(gene, lineage)`.
   - Use omics_palette(n_lineages) for lineage colors instead of gene colors.
   - Keep facet_wrap(~gene, scales = "free_y") for gene faceting.
   - Add lineage to the legend (position = "bottom").
   - Points also colored by lineage (not gene) when lineage is present.
5. Title changes to "Gene dynamics over pseudotime (multi-lineage)" when lineage present.
6. Params: add compare_lineages (default "true") -- when "false" and lineage exists, facet by gene+lineage instead of overlay.

Key constraint: The enhancement must be backward compatible. If gene_expression.csv has no lineage column, output is identical to current behavior.

**Part B: Add plot_ccc_bipartite to communication.R**

Ligand-focused bipartite layout (CCCNetworkPlot bipartite equivalent from scop). Shows sender cell types on the left, a central ligand-receptor pair column, and receiver cell types on the right, with edges connecting through the LR pair.

CSV input: top_interactions.csv (already exported by sc-cell-communication). Required columns: source, target, ligand, receptor, score.

Implementation approach (adapted from scop bipartite_plot):

1. Read top_interactions.csv. Require columns: source, target, ligand, receptor, score.
2. Select the focus ligand: params[["ligand"]] or auto-detect as the ligand with highest total score.
3. Filter to rows with that ligand. Keep top_n edges (default 20, by score).
4. Build 3-column layout:
   - Left column (x=0): unique sender cell types, evenly spaced vertically.
   - Center column (x=0.5): ligand node at top, receptor nodes below.
   - Right column (x=1): unique receiver cell types, evenly spaced vertically.
5. Draw edges with geom_segment or geom_curve:
   - sender -> ligand edges (left to center-top)
   - receptor -> receiver edges (center-bottom to right)
6. Node styling: cell type nodes = circles colored by omics_palette. LR nodes = diamonds/squares.
7. Edge width proportional to score. Edge alpha = 0.6.
8. Labels: cell type names next to nodes, LR names at center.
9. Params: ligand (default auto), top_n (default 20).
10. Use coord_fixed() + theme_void() base, then add title.

**Part C: Add plot_ccc_diff_network to communication.R**

Differential CCC network between conditions (CCCNetworkPlot diff_network equivalent). Shows changes in cell-cell communication between two conditions as a network with edges colored by direction of change (up/down).

CSV input: NEW file `ccc_diff_network.csv` in figure_data/. If this file does not exist, fall back to trying `sender_receiver_summary.csv` with a "condition" column. Required columns: source, target, score_diff (positive = increased, negative = decreased). Optional: condition_a, condition_b.

If no diff data available at all, print WARNING and exit gracefully (quit status=0).

Implementation approach:

1. Try to read ccc_diff_network.csv. Fallback: if sender_receiver_summary.csv has "condition" column, compute diff from the two conditions.
2. Filter to edges with |score_diff| > min_diff (default 0). Remove self-loops.
3. Arrange nodes in a circle (same layout as existing plot_ccc_network).
4. Draw edges: geom_curve with color mapped to sign of score_diff (red = increased, blue = decreased). Width proportional to |score_diff|.
5. Nodes: circles colored by cell type using omics_palette. Size proportional to total |score_diff|.
6. Labels: cell type names outside circle.
7. Legend: "Increased" (red) / "Decreased" (blue) for edge direction.
8. Params: min_diff (default 0), top_n (default 20).
