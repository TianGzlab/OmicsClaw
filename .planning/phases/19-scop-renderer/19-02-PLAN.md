**Step A: Create correlation.R** (new file)

Create `skills/singlecell/_lib/viz/r/correlation.R` with one renderer: `plot_feature_cor`.

Implementation per RESEARCH Pattern 5 (simplified 2-feature scatter):
- Read `gene_expression.csv` from data_dir (long format: cell_id, gene, expression)
- Params: `features` (comma-separated, e.g. "TP53,BRCA1"), `cor_method` ("pearson" or "spearman", default "pearson")
- If `features` param not provided, take the first 2 unique genes from the CSV
- Pivot to wide format using `reshape()`: one column per gene, rows = cell_id
- Compute correlation: `cor(wide_df[[gene1]], wide_df[[gene2]], method = cor_method, use = "complete.obs")`
- Compute p-value: `cor.test(...)$p.value`
- Build scatter: `geom_point(alpha = 0.3, size = 1)` + `geom_smooth(method = "lm", se = TRUE, color = "#E41A1C")`
- Add correlation annotation in top-left corner: `annotate("text", label = sprintf("r = %.3f\np = %.2e", cor_val, p_val))`
- Title: paste0(gene1, " vs ", gene2, " correlation")
- Labels: x = gene1 expression, y = gene2 expression
- `theme_omics()` + `ggsave_standard(p, out_path, width = 7, height = 6)`
- Edge case: if fewer than 2 genes in CSV, stop with informative error
- Edge case: if `features` has more than 2 genes, use only the first 2 (note: full matrix deferred to Phase 20)
